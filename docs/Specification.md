# Specification: HEIC to JPEG Converter

## 1. システム概要
本アプリケーションは、Windows環境において、指定されたディレクトリ内の複数のHEICファイルを高速にJPEGへ一括変換する単一実行ファイル型のCLI（コマンドラインインターフェース）ツールである。
「圧倒的な変換速度」と「直感的なユーザー体験（フォルダ配置＋ダブルクリック）」を提供することを目的とする。
**前提条件として、実行環境にはWindowsの「HEIF 画像拡張機能」および「HEVC ビデオ拡張機能」がインストールされているものとする。**

## 2. アーキテクチャと処理フロー

### 2.1. 全体処理フロー
1. **起動と初期化**: `.exe`が実行されたカレントディレクトリ（実行ファイルが置かれている場所）を取得する。
2. **ファイル走査 (Scanner)**: 対象ディレクトリを**直下のみスキャンし（サブディレクトリは無視）**、拡張子が `.heic` または `.HEIC` のファイルをリストアップする。この際、**隠しファイルやシステムファイルは変換対象外としてスキップ**する。
3. **スキップ判定**: 各HEICファイルに対して、出力先となる `{フォルダ名}_output` フォルダ内に同名の `.jpg` ファイルが存在するかチェックし、存在する場合は変換対象から除外する。
4. **ジョブ振り分け (Dispatcher)**: 変換対象のファイルリストを、PCの論理コア数（またはGPUリソース）に合わせたワークキュー/スレッドプールに投入する。
5. **並列変換 (Workers)**: 複数のワーカースレッドが同時に以下の処理を行う。
   - **HEICデコード**: GPU利用可能な場合はハードウェアデコード、不可の場合はCPUデコード。
   - **JPEGエンコード**: 品質設定 `95` でJPEGフォーマットにエンコードする。
   - **メタデータ転送**: 元ファイルのExifデータ（撮影日時やGPS等）を抽出してJPEGに埋め込む。
6. **結果集計 (Logger)**: 成功・失敗・スキップの数をリアルタイムで集計し、コンソールにプログレス（進捗）を表示する。
7. **終了待機**: 処理完了後、全結果のサマリーを表示し、ユーザーがEnterキーを押すまでコンソールウィンドウを維持する。

## 3. モジュール構成と責務

- **Main / CLI Manager**:
  アプリの起点。OSからのシグナルハンドリング、初期設定、終了時の待機処理（Enterキー待ち）を担う。
- **File System / Scanner**:
  ディレクトリの読み込み、拡張子のフィルタリング、ファイルの存在確認（スキップ判定）、出力パス（`{フォルダ名}_output`）の生成を行う。
- **Conversion Engine (Core)**:
  - **Decoder**: `libheif`等のバインディングやOSネイティブAPI（WIC）を通じてHEIC画像をピクセルデータに展開する。
  - **Encoder**: 展開されたピクセルデータを `libjpeg-turbo` 等を用いて高速にJPEG化（品質: 95）する。
  - **GPU Accelerator**: 実行マシンのGPU環境（DirectX/WIC, OpenCL, または専用API）を検知し、エンコード/デコード処理をオフロードする機能を提供するロジック。
- **Metadata Handler**:
  元ファイルからExifタグを安全に抽出し、生成されるJPEGファイルにロスレスで注入・コピーする。処理が重くならないよう画像データとは独立して扱う設計とする。

## 4. エラーハンドリングと堅牢性
- **ファイルの破損**: HEICファイル自体が破損しておりデコードに失敗した場合は、クラッシュ（パニック）を回避し、内部エラーをキャッチした上で「失敗ファイルリスト」にパスを記録する。
- **書き込み権限エラー**: 対象フォルダに書き込み権限がない場合など、I/Oエラーが発生した際も同様に記録し、全体のプロセスは止めない。
- **排他制御とメモリ保護 (OOM対策)**: 巨大な画像を複数同時に展開する際のメモリ枯渇（Out of Memory）を防ぐため、同時に処理する最大並列数（スレッド数）にハードリミットを設ける。並列実行環境における共有リソース（進捗カウンター等）へのアクセスはミューテックスで保護する。

## 6. テクノロジースタックの選定と方針
前提条件である「WindowsのHEIC/HEVC拡張機能がインストール済みであること」を最大限に活かし、最高速の変換とハードウェアアクセラレーション（GPU）を実現するため、以下の技術スタックを採用する。

**採用技術: C# (.NET 8) + Windows Imaging Component (WIC) + Native AOT**
- **WIC (Windows Imaging Component) の採用**: OSネイティブの画像処理エンジンを直接呼び出す。これにより、拡張機能が提供するハードウェアレベル（GPUデコード）を超高速・低オーバーヘッドで利用することが可能になる。
- **.NET 8 と System.Threading.Tasks**: C# の `Parallel.ForEachAsync` 等を利用した、極めて効率的なマルチスレッド処理（タスク並列処理）を実装する。
- **Native AOT (Ahead-of-Time) コンパイル**: コンパイル時に.NETランタイムをすべて実行ファイル内に静的リンクし、ネイティブコードへ変換する。これにより起動時間の短縮とメモリ使用量の削減を実現し、「依存関係なしの単体高速 `.exe`」という要件を完璧に満たす。
