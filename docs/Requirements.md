# Requirements: HEIC to JPEG Converter

## 1. 概要
指定されたフォルダ内のHEIC形式の画像ファイルを、高速かつ効率的にJPEG形式に変換するWindows向けデスクトップアプリケーション（.exe）。
元のHEICファイルは削除せず保持する。最も重視する要件は「変換速度」である。

## 2. 機能要件
- **変換機能**: `.heic`ファイルを`.jpg`ファイルに変換する。
- **並列処理**: 複数のHEICファイルを同時並行で変換し、CPUリソースを最大限に活用して高速化を図る。
- **ファイル保持**: 変換元のHEICファイルは変換後も変更や削除を行わずにそのまま残す。
- **出力先**: 変換後のJPEGファイルは、元のHEICファイル群が入っているフォルダのフォルダ名に `_output` を付与した名前のフォルダ（例：元のフォルダが `Photos` なら `Photos_output` フォルダ）を同階層に作成し、その中に保存する。
- **対象形式**: 
  - 入力: `.heic`, `.HEIC`
  - 出力: `.jpg`
- **Exifメタデータの保持**: 変換後も元のHEICファイルに含まれるExifメタデータ（撮影日時、GPS情報など）をJPEGに引き継ぐこと。
- **ファイル名の同期**: 生成されたJPEGファイルのファイル名(拡張子を除く)を元のHEICファイルと完全に一致させること。
- **重複設定（同名ファイル処理）**: 既に変換先の同名JPEGファイルが存在する場合は、変換（上書き）をスキップして次の処理に進むこと。
- **画質設定**: JPEGの出力品質（圧縮率）は `95` とする。

## 3. 非機能要件
- **実行速度（最重要）**: マルチスレッド（並列処理）を活用し、大量のファイルを可能な限り短時間で変換すること。
- **GPUアクセラレーション**: 実行環境に利用可能なGPU（ハードウェアエンコーダ/デコーダ）が存在する場合は自動的にGPUを使用し、さらなる高速化を図ること。GPUが利用できない環境ではCPU処理へシームレスにフォールバックすること。
- **実行形式**: Windows環境で単体で動作する`.exe`ファイルであること。
- **依存環境**: ユーザー側でランタイムや外部ライブラリをインストールしなくても、単体のexeで動作する（または必要なDLLが同梱されている）こと。
- **エラー処理（堅牢性）**: 破損したHEICファイルなどで変換に失敗した場合は、アプリをクラッシュさせずにそのファイルをスキップし、次の処理を継続すること。全処理完了後に、成功件数・失敗件数とエラーとなったファイル名を一覧で出力すること。

## 4. ユーザー体験（UX）
- **起動方法**: アプリ（`.exe`）をHEICファイル群が存在するフォルダに移動し、ダブルクリックして起動するだけで、自動的に同一フォルダ内のHEICファイルを検知し、JPEGへの変換処理を開始すること。
- **進捗表示**: 変換中の進捗（何枚中何枚完了したか）が分かるシンプルな文字出力（コンソール表示）を行うこと。
- **結果の確認（自動終了防止）**: 処理完了後、またはエラー発生時にコンソール画面（黒い画面）が即座に閉じてしまわないよう、最後に「Enterキーを押して終了してください」などの入力待ち状態にすること。

## 5. 検討すべき技術アプローチ
速度を最優先するため、以下のいずれかのアプローチを使用する。
1. **Rust**: 最高速のパフォーマンス、安全なマルチスレッド処理、単一exeファイルの作成において最強候補。
2. **Go言語**: スレッド処理（Goroutine）が非常に簡単で高パフォーマンス、クロスコンパイルも容易。
